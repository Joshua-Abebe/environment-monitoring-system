MQTT Event Lifecycle

Sensor Simulator
      ↓
 MQTT Publish
      ↓
 Mosquitto Broker
      ↓
 Subscriber Engine
      ↓
 Validation Layer
      ↓
 Processing Layer
      ↓
 Database Routing
      ↓
 Alert Engine
      ↓
 Analytics System



 Sensor generation:
 - All sensors generate events asynchronously

 MQTT publishing:
 - Only know about broker's address and topic

 Broker distribution:
 - Doesn't underatand data, know meaning, nor store data permanently

 Subscriber ingerstion:
 - Writes directly to database -> MQTT CALLBACK (in the future we might create enqueue before wirting the sensor data directly to the database)

Validation layer:
- Ensuring this exist in the incoming message:
               - sensor_id
               - value
               - timestamp
- Value range checks:
               - Temperature: -20 to 80
               - Humidity: 0% to 100%
               - AQI: 0 to 500

- And any invalid messages will be logged

Processing layer:
- After validation, the data becomes trusted
- Store in:
       - MySQL (structured analytics storage)
       - MongoDB (raw event)
       - Neo4j (relationship intelligence, mostly dormant only during initialization)
- At system startup:
       - initialize rooms
       - initialize sensors
       - initialize graph relationship, after that Neo4j ramains mostly stable

Alert engine:
- After storing event, evaluate ris*

Analytics access:
- Querying + insight generation



Important Architectural Principle

Write path and Read path:

Write path:
- Sensors -> MQTT -> subscriber -> databases
- Ingestion, persistence, and validation
Read path
- Analytics engine -> databases -> reports
- Querying, aggregation, and intelligence

System startup sequence
- Order matters because:
                 - Subscriber can not store data before DBs initializes
                 - Sensor should not publish before subscriber is ready
                 - Graph relationships should exist before analytics run

1. Load configuration
2. Connect databases
3. Initialize MySQL tables
4. Initialize Neo4j graph
5. Start MQTT subscriber
6. Start sensor simulators
7. Begin analytics operations
